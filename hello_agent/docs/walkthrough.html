Walkthrough: Module 0 - Hello World Agent

This lab demonstrates Agency in action. You will watch an AI agent "think" through a problem, use tools, and make a decision that a regular script couldn't.

Prerequisites

  • Ollama running locally with the llama3 model.
  • Python 3.10+ (REQUIRED for CrewAI type compatibility).
  • Venv dependencies installed (run ./venv/bin/pip install -r ../requirements.txt if needed).

1. Running the Demo

Navigate to the module folder and run the script using the local virtual environment:

cd hello_agent
./venv/bin/python3 agent_demo.py

[!IMPORTANT] Why the venv? CrewAI requires Python 3.10+ features (like the | union type). Using the provided venv ensures you are running on Python 3.12 even if your system default is older.

2. What to Watch For (The "Click" Moments)

As the script runs, focus on the terminal output. You will see the Agentic Loop in real-time:

A. The "Thought" Process

The Agent doesn't just run code. It writes out its logic:

"I need to check the disk space first to confirm the error..."

B. Tool Use

Watch the agent choose the check_disk_space tool. It isn't forced to; it decides it needs that information to proceed.

C. Critical Reasoning (The Highlight)

The agent will see critical_config.json in the list. Because of its Backstory, it will decide:

"I identified temp_debug.log as trash, but I will ignore critical_config.json as it is vital for the server." A script would have deleted both.

3. New Advanced Features (Agentic Operations)

Now that you've seen the basic loop, notice the advanced capabilities we've added to demonstrate Enterprise-Ready Agency:

A. Observability (Agent Trace)

Your agent is no longer a "black box."

  • How to view: Open your Splunk dashboard (GenAI Observability).
  • What to see: Every thought, tool call, and model response from the junior_admin is now captured as an OpenTelemetry trace. You can precisely measure latency and token cost for each administrative decision.

B. Security (Cisco AI Defense Proxy)

Before the agent acts, its "Thought" is inspected for security violations.

  • The Proxy: Look for the [AI Defense Proxy] section in your terminal.
  • The Rule: If the agent attempts to handle sensitive data (like password or ssn), the proxy will block the execution before it reaches the model. This is your "AI Safety Guardrail."

C. Persistence (Agent Memory)

The agent now remembers its history.

  • Long-Term Memory: Look for the memory folder. Inside is a long_term_memory_storage.db.
  • The Benefit: If you run the script again, the agent will refer to its previous findings, leading to faster and more efficient problem-solving. It builds a "Knowledge Base" of your infrastructure issues.
